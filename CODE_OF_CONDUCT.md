# Code of Conduct

This project follows simple ground rules so that everyone — maintainers,
contributors, users — can collaborate productively.

## Expectations

- Be respectful and patient. People come from different backgrounds and
  skill levels.
- Focus on the technical content. Disagreements about code, design, or
  trade-offs are welcome; personal attacks are not.
- Assume good intent. If you suspect a misunderstanding, ask before
  escalating.
- Stay on topic. Issues and PRs are about clientctl; off-topic content
  will be redirected or closed.
- Don't share personal data of others without consent (e.g. screenshots
  with private paths, hostnames, tokens).

## Scope

These rules apply to all project spaces — issues, pull requests,
discussions, commit messages, and any other interaction tied to this
repository.

## Enforcement

The maintainer is the final arbiter. Consequences for violating these
rules range from a warning to being blocked from the repository,
proportional to the severity and pattern of behavior.

## Reporting

If something feels off, contact the maintainer directly via the email
on their GitHub profile rather than escalating in public threads.
Reports are handled confidentially.

This document is intentionally short. If you're looking for a more
formal framework, the [Contributor Covenant](https://www.contributor-covenant.org/)
is widely adopted and broadly compatible with the spirit of the rules
above.
